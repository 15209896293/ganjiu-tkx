## 三、API Key 替代方案

### 当前问题
`api.js` 中硬编码了 API Key。比赛提交后任何人都能看源码拿到Key。

### 方案：localStorage + 设置面板

**效果：**
- 首次进入AI实验室，弹出小面板让用户填入自己的DeepSeek API Key
- Key 存入浏览器 localStorage，以后自动读取
- 源码中不包含任何真实Key

**实现思路（给VSCode DeepSeek的改动点）：**

```
api.js 改造：
  - API_CONFIG.apiKey 默认值改为从 localStorage 读取
  - 新增函数 getApiKey()：优先读 localStorage → 没有则返回空字符串
  - 新增函数 setApiKey(key)：写入 localStorage

index.html 改造：
  - 在AI实验室站（station3）顶部加一个折叠式"⚙️ API设置"小面板
  - 包含：输入框（type=password遮盖）、保存按钮、状态提示

main.js 改造：
  - submitLab() 调用前先检查 getApiKey() 是否为空
  - 为空则展开设置面板，提示用户填入Key，不发送请求
```

### 🟡 任务C：localStorage 进度持久化

````
当前刷新页面后答题进度和徽章全丢失。实现自动保存和恢复。

【改 main.js】
1. 新增存储函数：
   function saveProgress() {
     localStorage.setItem('aijourney_progress', JSON.stringify({
       quizIndex: STATE.quizIndex,
       badges: STATE.badges,
       preScore: STATE.preScore,
       postScore: STATE.postScore,
       aiLabUsed: STATE.aiLabUsed,
       timestamp: new Date().toISOString()
     }));
   }
   
   function loadProgress() {
     var saved = localStorage.getItem('aijourney_progress');
     if (saved) {
       var data = JSON.parse(saved);
       STATE.quizIndex = data.quizIndex || 0;
       STATE.badges = data.badges || [];
       STATE.preScore = data.preScore || 0;
       STATE.postScore = data.postScore || 0;
       STATE.aiLabUsed = data.aiLabUsed || false;
       return true;
     }
     return false;
   }

2. 在以下时机调用 saveProgress()：
   - 答题答对/答错后
   - 徽章解锁后
   - AI实验室使用后

3. initApp() 开始时调用 loadProgress() 恢复状态

4. 在首页加一个小提示："📝 上次进度已恢复"（有存档时显示）

5. 在资源库站底部加"🗑️ 清除所有进度"按钮，点击后确认再清除

改完后测试：答几道题→刷新→进度应该还在。
````
